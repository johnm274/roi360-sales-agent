# Implementation — Running a Successful Deployment

## The Proof of Concept (POC) Approach

Always start with a POC. Don't try to build everything at once. A focused pilot proves value quickly and builds momentum.

### POC Structure
- **Duration:** 3-6 months
- **Scope:** 3-5 templates covering the client's highest-volume, highest-pain materials
- **Users:** Start with one department, region, or group of power users (10-50 people)
- **Success criteria:** Defined upfront (e.g., "80% of business card orders placed through the portal within 2 months")
- **Price:** Full list price — no discounts for POCs

### POC Template Selection
Choose templates that will have **immediate, visible impact**:
1. **Business cards** — everyone needs them, high frequency, easy win
2. **Letterheads / compliment slips** — used daily, brand consistency matters
3. **Leaflets or flyers** — frequently requested, currently bottleneck
4. **Posters / notices** — branch-level use, often done badly in Word
5. **Social media templates** — growing demand, brand risk if done ad-hoc

### What NOT to Start With
- Complex variable data pieces (annual reports, catalogues)
- Low-frequency items (once-a-year event materials)
- Materials that require extensive custom design each time
- Items with many stakeholders or complex approval chains

## Project Roles

### Your Side (Partner)
| Role | Responsibility |
|------|---------------|
| **Account Manager** | Client relationship, commercial, check-ins |
| **Project Lead** | Configuration, template building, testing |
| **Production** | Print integration, fulfilment setup |

### Client Side
| Role | Responsibility |
|------|---------------|
| **Project Sponsor** | Signs off scope, removes blockers, champions internally |
| **Brand/Marketing Contact** | Provides brand assets, approves templates, defines rules |
| **IT Contact** (if needed) | SSO setup, network access, security review |
| **Pilot Users** | First users, provide feedback, become internal advocates |

## Implementation Timeline

### Week 1-2: Setup & Discovery
- Gather brand assets (logos, colours, fonts, guidelines)
- Confirm template list and prioritise
- Define user groups, roles, and permissions
- Configure the storefront (branding, settings, workflows)

### Week 3-4: Template Build
- Build templates from client artwork or create new designs
- Configure editable vs. locked fields
- Set up variable data connections if needed
- Internal review and testing

### Week 5: Client Review & Training
- Present templates to client for feedback and approval
- Make amendments
- Train admin users (how to manage users, view reports)
- Train end users (how to order, customise, approve)

### Week 6: Go-Live
- Soft launch with pilot group
- Monitor usage and gather feedback
- Address any issues quickly
- Send "getting started" communications to users

### Month 2-3: Review & Optimise
- Review usage data with client
- Identify high-performing templates and areas for improvement
- Address adoption gaps (who's not using it? why?)
- Plan expansion: more templates, more users, more departments

## Early Wins

Focus on generating quick, visible wins in the first 2 weeks after go-live:

- **"First order in 5 minutes"** — get a user to place their first order and celebrate the speed
- **"No more email chains"** — show the client how many requests are now self-service
- **"Brand-perfect every time"** — compare an old off-brand example with the new templated version
- **"Admin visibility"** — show the client their dashboard with real usage data

## Common Pitfalls

1. **Too many templates too soon** — overwhelms users, slows the project, increases cost. Start with 3-5.
2. **No internal champion** — if no one at the client is pushing adoption, it stalls. Identify your champion early.
3. **Poor template design** — if templates are hard to use or don't look good, users won't come back. Invest in quality.
4. **Skipping training** — "it's intuitive" isn't enough. A 30-minute group session makes a huge difference.
5. **No follow-up after go-live** — the first month is critical. Check in weekly, review usage, act on feedback.
6. **Trying to replace the designer** — the platform handles routine work. Complex, bespoke design still needs a human. Position it as "freeing your designer to do strategic work."
